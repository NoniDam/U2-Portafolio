Introducción al Desarrollo Web Orientado a Servicios

Tema 1. Paradigma del desarrollo de aplicaciones orientadas a servicios

Servicios que se ofrecen en la nube

La computación en la nube permite acceder a recursos tecnológicos a través de Internet sin necesidad de contar con infraestructura física propia. Estos servicios facilitan el desarrollo, despliegue y uso de aplicaciones de forma flexible y escalable.

Características de las aplicaciones orientadas a servicios

Las aplicaciones orientadas a servicios se construyen a partir de servicios independientes que pueden comunicarse entre sí. Esto permite reutilizar funcionalidades, mejorar la interoperabilidad entre sistemas y facilitar el mantenimiento.

Aplicaciones web híbridas (Mashup)

Las aplicaciones web híbridas combinan información y servicios provenientes de distintas fuentes para crear nuevas funcionalidades. Son comunes en aplicaciones modernas que integran mapas, datos externos o servicios de terceros.

Tema 2. Arquitectura Orientada a Servicios (SOA)

¿Qué es la Arquitectura Orientada a Servicios (SOA)?

SOA es un modelo de arquitectura de software que organiza las aplicaciones como un conjunto de servicios bien definidos, independientes y reutilizables, los cuales se comunican mediante protocolos estándar.

Principios de diseño en SOA

Los servicios en SOA siguen principios como bajo acoplamiento, reutilización, interoperabilidad y escalabilidad, lo que permite que los sistemas sean más flexibles y fáciles de adaptar a nuevos requerimientos.

Estándares utilizados en los servicios web

Los servicios web emplean estándares que aseguran una comunicación eficiente entre aplicaciones, independientemente del lenguaje o plataforma utilizada.

Estándares relacionados con los servicios web

XML

Lenguaje utilizado para estructurar y transportar datos de manera legible tanto para humanos como para máquinas.

SOAP

Protocolo que define cómo se intercambian mensajes estructurados entre servicios web.

WSDL

Documento que describe las operaciones, métodos y forma de acceso a un servicio web.

UDDI

Registro que permite publicar y localizar servicios web disponibles.

REST

Arquitectura ligera basada en HTTP que facilita la creación de servicios simples y escalables.
